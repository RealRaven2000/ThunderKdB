ImportExportTools NG Обзор и меню

ImportExportTools NG - это дополнительное расширение, основанное на оригинальной работе Паоло "Kaosmos". Расширение предлагает несколько функций импорта и экспорта, поддерживающих несколько различных форматов файлов, включая: Примечание. Некоторые форматы поддерживаются не всеми функциями.
- mbox
- EML
- maildir
- HTML
- простой текст
- CSV (значения, разделенные запятыми)
- PDF

В то время как большая часть функциональности вращается вокруг сообщений, есть также поддержка импорта и экспорта профилей Thunderbird. Кроме того, экспорт также может включать индексный файл HTML или CSV. Резервное копирование профиля Thunderbird - еще одна особенность.

Доступ к функциональности IETng осуществляется через несколько различных меню, установленных расширением. Функции являются контекстно-зависимыми. Меню добавляются для:
- Контекст панели папки (операция с папкой)
- Контекст списка сообщений (выбранные сообщения)
- Меню Инструменты (операции с папками и профилями)

Существует много опций, которые доступны в добавленной записи в разделе «Инструменты / Опции надстроек» для опций расширения. Каждое из меню также имеет пункт меню «Параметры» для доступа к диалоговому окну параметров.
Новые параметры документированы здесь, весь набор будет документирован впоследствии.

Пользовательский формат даты

Пользовательская опция даты позволяет полностью контролировать форматирование даты, используемое для обоих файлов в индексе. Как формат по умолчанию «Дата», так и новый пользовательский формат «Дата (пользовательский)» доступны в раскрывающихся списках имен файлов. Настраиваемую дату также можно использовать с токеном $ {date_custom}, используемым в расширенных настраиваемых именах файлов. Исходный (по умолчанию) формат даты:
ГГГГММДДЧЧмм: 202011060808

Пользовательский формат даты использует% последовательностей, соответствующих функциональности, подобной Unix strftime() . Он основан на порте JavaScript strftime() , функции для форматирования даты и времени.
Порт strftime () от TH Doan (https://thdoan.github.io/strftime/)

%b Сокращенное название месяца.

%d День месяца в виде десятичного числа (от 01 до 31).
%e День месяца в виде десятичного числа (от 1 до 31).
%F Формат даты ISO 8601 (эквивалент %Y-%m-%d ).
%G ISO 8601 недельный год со столетием в виде десятичного числа. Год из 4 цифр соответствует номеру недели ISO (см. %V ). Он имеет тот же формат и значение, что и %Y , за исключением того, что если номер недели ISO относится к предыдущему или следующему году, вместо этого используется этот год.
%g Как %G , но без столетия, то есть с двузначным годом (00-99).
%H Час в виде десятичного числа с использованием 24-часовых часов (от 00 до 23). Смотрите также %k .
%I Час в виде десятичного числа с использованием 12-часовых часов (диапазон от 01 до 12). Смотрите также %l .
%j День года в виде десятичного числа (от 001 до 366).
%k Час в виде десятичного числа с использованием 24-часовых часов (от 0 до 23). Смотрите также %H
%l Час в виде десятичного числа с использованием 12-часовых часов (от 1 до 12). Смотрите также %I
%m Месяц в виде десятичного числа (от 01 до 12).
%n Месяц в виде десятичного числа (от 1 до 12).
%M Минута в виде десятичного числа (от 00 до 59).
%p Либо «AM», либо «PM» в соответствии с заданным значением времени. Полдень рассматривается как «PM», а полночь - «AM».
%P Как %p но в нижнем регистре («am» или «pm»).
%s Количество секунд с начала эпохи, 1970-01-01 00:00:00 +0000 (UTC).
%S Второй как десятичное число (от 00 до 59).
%u День недели десятичный (от 1 до 7), понедельник - 1. См. Также %w .
%V Номер недели ISO 8601 текущего года в виде десятичного числа (в диапазоне от 01 до 53), где неделя 1 - это первая неделя, в которой не менее 4 дней в новом году (то есть первый четверг).
%w День недели в десятичном формате (от 0 до 6), воскресенье - 0. Смотрите также %u .
%x Предпочтительное представление даты для текущей локали без времени.
%X Предпочтительное представление времени для текущей локали без даты.
%y Год как десятичное число без столетия (от 00 до 99).
%Y Год как десятичное число, включая век.
%z Числовой часовой пояс +hhmm или -hhmm (то есть смещение часов и минут от UTC).
%Z Название часового пояса или сокращение.

Примечания совместимости:

  • %c - форматированная строка немного отличается
  • %D - не реализовано (используйте %m/%d/%y или %d/%m/%y )
  • %e - пробел не добавлен
  • %E - не реализовано
  • %h - не реализовано (используйте %b )
  • %k - пробел не добавлен
  • %n - как %m , но без начального нуля (используйте \n для новой строки)
  • %O - не реализовано
  • %r - не реализовано (используйте %I:%M:%S %p )
    • ${subject} Тема сообщения
  • %R - не реализовано (используйте %H:%M )
  • %t - не реализовано (используйте \t )
  • %T - не реализовано (используйте %H:%M:%S )
  • %U - не реализовано
  • %W - не реализовано
  • %+ - не реализовано
  • %% - не реализовано (используйте % )


Расширенный формат имени файла:


Расширенное форматирование имени файла использует токены. Точное имя файла может быть создано с использованием токенов, а также любого текста или допустимых символов имени файла. Английские (en-US) токены доступны в любой локали. Также доступны специальные токены для локали, которые перечислены под английской таблицей.

Примеры:

${subject}-${date}-${index} - формат по умолчанию
${prefix}_${date_custom}_${sender} - использовать пользовательский формат даты вместо значения по умолчанию
${subject}-GmailMsg-${date}-${index} - с произвольным текстом


знак Описание
${subject} Тема сообщения. Длина может быть ограничена параметром длины объекта.
${sender} Имя отправителя сообщения.
${recipient} Имя получателя сообщения.
${smart_name} Получатели, если сообщение находится в папке «Отправленные», в противном случае имя отправителя
${index} Индексный номер сообщения.
${prefix} Значение параметра префикса текста.
${suffix} Значение параметра текста суффикса.
${date} Стандартный формат даты. Формат по умолчанию: ГГГГММДДЧЧмм
${date_custom} Пользовательский формат даты. Смотрите пользовательский раздел форматирования даты.

Новые и экспериментальные настройки (варианты)

Есть несколько новых функций и опций, которые еще не настраиваются в диалоге опций. Как только эти функции стабилизируются, они будут добавлены в пользовательский интерфейс.
Для их использования они должны быть установлены в редакторе конфигурации. Чтобы получить доступ к редактору, используйте меню «Инструменты / Опции», затем выберите раздел «Дополнительно», а внизу будет кнопка «Редактор конфигурации». Если вы введете «importexporttoolsng» в строке поиска, вы получите все настройки расширения. Затем вы можете отредактировать различные настройки ниже по мере необходимости.

importexporttoolsng.export.index_date_custom_format

Это предпочтение следует тому же полному форматированию, которое будет использоваться для индексных файлов. По умолчанию это пустая строка, если с форматированной строкой даты / времени, это изменит формат даты индекса.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Оба эти предпочтения используют тот же тип расширенных форматов именования файлов, который используется в экспортируемом сообщении с именем файла. Первый предназначен для внешних вложений, второй - для встроенных (встроенных) вложений.

importexporttoolsng.experimental.use_delivery_date

Обычно «Дата» для сообщения в Thunderbird - это дата, установленная почтовым клиентом отправителя. Установка этой опции в «истину» использует полученную дату в почтовом клиенте получателя. Эта дата будет использоваться как для имен файлов, так и для даты в индексных файлах.

Базовая, глобальная горячая клавиша (ярлыки)

importexporttoolsng.experimental.hot_keys

Это новое предпочтение позволяет пользователю установить до 10 сочетаний клавиш. Ярлыки могут выполнять большинство функций ImportExportTools NG, а также другие функции Thunderbird. Строка предпочтения отформатирована как объект в формате JSON. Формат и ключи следующие:

Пример записей горячих клавиш (массив объектов JSON)

id - индекс ключа (1-10)
key - ключ (одиночный символ)
keycode - определяющий keycode VK_ *
Может быть включен ключ или код ключа, но не оба
modifiers - ключевые модификаторы, разделенный пробелами список ('control', 'shift', 'alt', 'accel')
oncommand - прямая командная строка
contexts - contexts окна, в котором активен ключ
'all' - все окна в Messenger, создание, поиск
'messenger' - главное окно посланника и его потомки
'compose' - окно создания сообщения
'search' - диалог поиска

[{"id": "1", "key": "K", "modifiers": "control shift", "oncommand": "alert('Compose alert')", "contexts": "compose"}, {"id": "2", "keycode: "VK_F1", "modifiers": "control shift", "oncommand": "alert('testD')", "contexts": "all"}]
