Visión xeral e menús de ImportExportTools NG

ImportExportTools NG é unha extensión seguida baseada no traballo orixinal de Paolo "Kaosmos". A extensión ofrece varias funcións de importación e exportación que admiten varios formatos de ficheiros diferentes, incluíndo: Nota: Algúns formatos non son compatibles en todas as funcións.
- mbox
- EML
- maildir
- HTML
- texto plano
- CSV (valores separados por coma)
- PDF

Aínda que a maior parte das funcións para xirar en torno ás mensaxes, tamén hai soporte para a importación e exportación de perfís Thunderbird. Ademais, as exportacións poden incluír un ficheiro de índice HTML ou CSV. A copia de seguridade do perfil Thunderbird é outra característica.

O acceso á funcionalidade IETng faise mediante varios menús diferentes instalados pola extensión. As funcións son sensibles ao contexto. Engádense menús para:
- O contexto do panel de carpetas (operación de cartafoles)
- O contexto da lista de mensaxes (mensaxes seleccionadas)
- O menú Ferramentas (operacións de cartafol e perfil)

Hai moitas opcións accesibles baixo a entrada engadida baixo a opción "Ferramentas / Opcións de complemento" para as opcións de extensión. Cada un dos menús tamén ten un elemento de menú Opcións para acceder ao diálogo de opcións.
Aquí están documentadas as novas opcións, o conxunto completo documentarase posteriormente.

Formato de data personalizado

A opción de data personalizada permite o control completo sobre a formato de data empregada para os dous ficheiros do índice. Tanto o formato predeterminado "Data" como o novo formato personalizado "Data (personalizado)" están dispoñibles no menú despregable. A data personalizada tamén se pode usar co token $ {date_custom} usado nos nomes de ficheiro personalizados estendidos. O formato de data orixinal (por defecto) é:
YYYYMMDDHHmm: 202011060808

O formato de data personalizado utiliza secuencias% correspondentes a strftime() Unix como función. Baséase nun porto JavaScript de strftime() , unha función para dar formato á data e hora.
Port of strftime () por TH Doan (https://thdoan.github.io/strftime/)

Descricións da secuencia de conversións

Secuencia Descrición
%a Nome abreviado do día da semana.
%A Nome completo do día da semana.
%b Nome do mes abreviado.
%B Nome completo do mes.
%c Representación de data e hora (UTC) preferida para a configuración local actual.
%C Número de século (ano / 100) como un número enteiro de 2 díxitos.
%d Día do mes como número decimal (intervalo 01 a 31).
%e Día do mes en número decimal (rango 1 a 31).
%F Formato de data ISO 8601 (equivalente a %Y-%m-%d ).
%G ISO 8601 ano baseado na semana con século como número decimal. O ano de 4 díxitos corresponde ao número de semana ISO (ver %V ). Este ten o mesmo formato e valor que %Y , excepto que se o número da semana ISO pertence ao ano anterior ou ao ano seguinte, úsase ese ano.
%g Como %G , pero sen século, é dicir, cun ano de 2 díxitos (00-99).
%H Hora como un número decimal empregando un reloxo de 24 horas (rango de 00 a 23). Vexa tamén %k .
%I Hora como un número decimal empregando un reloxo de 12 horas (intervalo 01 a 12). Vexa tamén %l .
%j Día do ano como número decimal (rango 001 a 366).
%k Hora como un número decimal empregando un reloxo de 24 horas (rango de 0 a 23). Vexa tamén %H
%l Hora como un número decimal empregando un reloxo de 12 horas (rango 1 a 12). Vexa tamén %I
%m Mes como número decimal (rango 01 a 12).
%n Mes como número decimal (rango 1 a 12).
%M Minuto como número decimal (rango 00 a 59).
%p Calquera "AM" ou "PM" segundo o valor do tempo dado. O mediodía é tratado como "PM" e a medianoite como "AM".
%P Como %p pero en minúsculas ("am" ou "pm").
%s Número de segundos desde a Epoca de 1970-01-01 00:00:00 +0000 (UTC).
%S Segundo como un número decimal (rango de 00 a 59).
%u Día da semana en decimal (rango 1 a 7), luns sendo 1. Véxase tamén %w .
%V Número ISO 8601 da semana actual como número decimal (rango 01 a 53), onde a semana 1 é a primeira semana que ten polo menos 4 días no ano novo (é dicir, o primeiro xoves).
%w Día da semana en decimal (rango de 0 a 6), sendo domingo 0. Ver tamén %u .
%x Representación de data preferida para a configuración local actual sen hora.
%X Representación horaria preferida para a configuración local actual sen data.
%y Ano como número decimal sen século (rango de 00 a 99).
%Y Ano como número decimal incluído o século.
%z A zona horaria numérica +hhmm ou -hhmm (é dicir, a compensación horaria e hora desde UTC).
%Z Nome ou abreviatura da zona horaria.

Notas de compatibilidade:

  • %c cadea con formato %c é lixeiramente diferente
  • %D : non implementado (uso %m/%d/%y ou %d/%m/%y )
  • Non se engadiu %e espazo
  • %E : non se implementou
  • %h - non implementado (uso %b )
  • %k - o espazo non se engadiu
  • %n - como %m , pero ningún cero principal (usa \n para a liña nova)
  • %O : non se implementou
  • %r : non implementado (uso %I:%M:%S %p )
    • ${subject} O asunto da mensaxe
  • %R : non implementado (uso %H:%M )
  • %t : non implementado (usa \t )
  • %T : non implementado (uso %H:%M:%S )
  • %U : non implementado
  • %W : non implementado
  • %+ : non se implementou
  • %% : non implementado (uso % )

Preferencias novas e experimentais (opcións)

Hai varias novas funcións e opcións que aínda non se poden configurar no diálogo de opcións. Unha vez estabilizadas estas funcións, engadiranse á interface de usuario.
Para usalos deben establecerse no Editor de configuración. Para acceder ao editor, use o menú Ferramentas / Opcións e seleccione a sección Avanzada e haberá un botón 'Config Editor' na parte inferior. Se escribe 'importexporttoolsng' na barra de busca obterás todas as preferencias de extensión. Podes editar as distintas preferencias a continuación, segundo sexa necesario.

importexporttoolsng.export.index_date_custom_format

Esta preferencia segue o mesmo formato completo que se emprega para os ficheiros de índice. Por defecto, é unha cadea baleira, se hai unha data / hora formateada, este cambiará o formato da data do índice.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

Ambas as preferencias usan o mesmo tipo de formatos de nome de ficheiros estendidos que usa o nome de ficheiro de mensaxes exportadas. O primeiro é para ficheiros adxuntos externos, o segundo é para arquivos adxuntos incrustados.

importexporttoolsng.experimental.use_delivery_date

Normalmente a 'Data' para unha mensaxe en Thunderbird é a data establecida polo cliente de correo do remitente. Se establece esta opción como "verdadeira" utiliza a data recibida no cliente de correo do destinatario. Esta data usarase tanto para nomes de ficheiros como para a data nos ficheiros de índice.

Función básica, global de tecla rápida (atallos)

importexporttoolsng.experimental.hot_keys

Esta nova preferencia permite ao usuario configurar ata 10 atallos de teclado. Os atallos poden executar a maioría das funcións ImportExportTools NG así como outras funcións de Thunderbird. A cadea de preferencia está formateada como un obxecto con formato JSON. O formato e as teclas son os seguintes:

Exemplo de entradas de chave en directo (matriz de obxectos JSON)

id - o índice de clave (1-10)
key - a chave (personaxe único)
keycode : o código de clave "VK_ *" defind
Pode incluírse calquera clave ou clave, pero non ambas
modifiers : os modificadores de chaves, lista separada por espazo ("control", "shift", "alt", "accel")
oncommand : unha cadea de comando directo
contexts : o contexts da xanela onde a chave está activa
'all' : todas as fiestras en mensaxeiro, compoñer, buscar
'messenger' : a xanela principal do mensaxeiro e os seus descendentes
'compose' : a xanela de compoñer a mensaxe
'search' : o diálogo de busca
