ImportExportTools NG 개요 및 메뉴

ImportExportTools NG는 Paolo "Kaosmos"의 원래 작업을 기반으로 한 후속 확장입니다. 이 확장은 다음과 같은 여러 가지 파일 형식을 지원하는 여러 가져 오기 및 내보내기 기능을 제공합니다. 참고 : 일부 형식은 일부 기능에서 지원되지 않습니다.
-mbox
-EML
-maildir
-HTML
- 일반 텍스트
-CSV (쉼표로 구분 된 값)
-PDF

메시지를 중심으로하는 대부분의 기능은 Thunderbird 프로필 가져 오기 및 내보내기를 지원합니다. 또한 내보내기에는 HTML 또는 CSV 색인 파일이 포함될 수도 있습니다. Thunderbird 프로필 백업은 또 다른 기능입니다.

IETng 기능에 액세스하려면 확장 프로그램에서 설치 한 여러 가지 메뉴를 통해 수행하십시오. 기능은 상황에 따라 다릅니다. 메뉴가 추가되었습니다 :
-폴더 창 컨텍스트 (폴더 작업)
-메시지 목록 컨텍스트 (선택된 메시지)
-도구 메뉴 (폴더 및 프로필 작업)

확장 옵션에 대한 "도구 / 애드온 옵션"아래의 추가 항목에서 액세스 할 수있는 옵션이 많이 있습니다. 각 메뉴에는 옵션 대화 상자에 액세스 할 수있는 옵션 메뉴 항목도 있습니다.
새로운 옵션은 여기에 문서화되어 있으며, 전체 세트는 이후에 문서화됩니다.

맞춤 날짜 형식

사용자 정의 날짜 옵션을 사용하면 인덱스의 두 파일 모두에 사용되는 날짜 형식을 완전히 제어 할 수 있습니다. 파일 이름 드롭 다운에서 기본 형식 인 "날짜"와 새로운 사용자 지정 형식 인 "날짜 (사용자 지정)"를 모두 사용할 수 있습니다. 맞춤 날짜는 확장 맞춤 파일 이름에 사용 된 $ {date_custom} 토큰과 함께 사용할 수도 있습니다. 원래 (기본) 날짜 형식은 다음과 같습니다.
YYYYMMDDHHmm : 202011060808

사용자 정의 날짜 형식은 strftime() Unix와 유사한 기능과 일치하는 % 시퀀스를 사용합니다. 날짜 및 시간을 형식화하는 함수 인 JavaScript 포트 strftime() 기반으로합니다.
TH Doan의 strftime () 포트 (https://thdoan.github.io/strftime/)

변환 순서 설명

순서 기술
%a 요일의 약칭.
%A 요일의 이름입니다.
%b 약식 월 이름.
%B 월 이름.
%c 현재 로캘의 기본 날짜 및 시간 (UTC) 표현
%C 2 자리 정수로 된 세기 숫자 (년 / 100).
%d 월의 일을 10 진수로 표시합니다 (01-31 범위).
%e 월의 일을 10 진수로 표시합니다 (1-31 범위).
%F ISO 8601 날짜 형식 ( %Y-%m-%d 와 동일).
%G 세기를 10 진수로 한 ISO 8601 주별 연도. 4 자리 연도는 ISO 주 번호에 해당합니다 ( %V 참조). ISO 주 번호가 이전 또는 내년에 속하는 경우 해당 연도가 대신 사용된다는 점을 제외하고는 %Y 와 형식 및 값이 같습니다.
%g %G 와 비슷하지만 세기가 없으면 2 자리 연도 (00-99)입니다.
%H 24 시간제 (00-23 범위)를 사용하는 10 진수로 된 시간입니다. %k 도 참조하십시오.
%I 12 시간제 (01-12 범위)를 사용하는 시간을 10 진수로 나타냅니다. %l 도 참조하십시오.
%j 연도를 10 진수로 표시합니다 (001-366 범위).
%k 24 시간제 (0-23 범위)를 사용하는 10 진수로 된 시간입니다. %H 도 참조하십시오.
%l 12 시간제 (1-12 범위)를 사용하는 10 진수로 된 시간입니다. %I 도 참조하십시오.
%m 십진수로 된 월 (01-12 범위).
%n 월을 10 진수로 표시합니다 (1 ~ 12 범위).
%M 십진수로 된 분 (00-59 범위).
%p 주어진 시간 값에 따라 "AM"또는 "PM"입니다. 정오는 "PM"으로 자정은 "AM"으로 처리됩니다.
%P %p 비슷하지만 소문자 ( "am"또는 "pm")입니다.
%s Epoch, 1970-01-01 00:00:00 +0000 (UTC) 이후의 시간 (초)입니다.
%S 두 번째는 10 진수 (00 ~ 59 범위)입니다.
%u 요일은 10 진수 (1-7 범위)이며 월요일은 1입니다. %w 도 참조하십시오.
%V 현재 연도의 ISO 8601 주 번호는 10 진수 (01 ~ 53 범위)로 표시됩니다. 여기서 주 1은 새해에 4 일 이상인 첫 번째 주 (첫 번째 목요일)입니다.
%w 요일을 10 진수 (0-6 범위)로, 일요일은 0입니다. %u 도 참조하십시오.
%x 시간이없는 현재 로캘의 기본 날짜 표시
%X 날짜가없는 현재 로캘의 기본 시간 표시
%y 연도는 세기가없는 10 진수 (00-99 범위)입니다.
%Y 세기를 포함하여 10 진수로 표현 된 연도입니다.
%z +hhmm 또는 -hhmm 숫자 시간대 (즉, UTC에서시 및 분 오프셋)입니다.
%Z 시간대 이름 또는 약어.

호환성 메모 :

  • %c 형식화 된 문자열이 약간 다릅니다
  • %D 구현되지 않음 ( %m/%d/%y 또는 %d/%m/%y )
  • %e 공백이 추가되지 않습니다
  • %E 구현되지 않음
  • %h 구현되지 않음 ( %b 사용)
  • %k 공백이 추가되지 않습니다
  • %n %m 과 비슷하지만 앞에 0이 없습니다 (개행에는 \n 사용).
  • %O 구현되지 않음
  • %r 구현되지 않음 ( %I:%M:%S %p )
    • ${subject} 메시지의 주제
  • %R 구현되지 않음 ( %H:%M )
  • %t 구현되지 않음 ( \t 사용)
  • %T 구현되지 않음 ( %H:%M:%S )
  • %U 구현되지 않음
  • %W 구현되지 않음
  • %+ -구현되지 않음
  • %% -구현되지 않음 ( % 사용)


확장 파일 이름 형식 :


확장 파일 이름 형식은 토큰을 사용합니다. 정확한 파일 이름은 토큰이나 텍스트 또는 유효한 파일 이름 문자를 사용하여 구성 할 수 있습니다. 영어 (en-US) 토큰은 모든 로케일에서 사용할 수 있습니다. 로케일 특정 토큰도 사용 가능하며 영어 테이블 아래에 나열되어 있습니다.

예 :

${subject}-${date}-${index} -기본 형식
${prefix}_${date_custom}_${sender} -기본값 대신 사용자 정의 날짜 형식을 사용하십시오.
${subject}-GmailMsg-${date}-${index} -일부 맞춤 텍스트 포함


토큰 기술
${subject} 메시지의 제목 길이는 피사체 길이 옵션에 의해 제한 될 수 있습니다.
${sender} 메시지 발신자의 이름입니다.
${recipient} 메시지 수신자의 이름입니다.
${smart_name} 메일이 보낸 편지함에 있으면 수신자, 그렇지 않으면 발신자 이름
${index} 메시지 색인 번호
${prefix} 접두사 텍스트 옵션 값입니다.
${suffix} 접미사 텍스트 옵션 값입니다.
${date} 표준 날짜 형식입니다. 기본 형식은 YYYYMMDDHHmm입니다.
${date_custom} 맞춤 날짜 형식 사용자 정의 날짜 형식 섹션을 참조하십시오.

토큰 (en-US) 현지화 ( ) 토큰
${subject}
${sender}
${recipient}
${smart_name}
${index}
${prefix}
${suffix}
${date}
${date_custom}

새로운 실험 환경 설정 (옵션)

옵션 대화 상자에서 아직 구성 할 수없는 몇 가지 새로운 기능과 옵션이 있습니다. 이러한 기능이 안정화되면 UI에 추가됩니다.
이를 사용하려면 구성 편집기에서 설정해야합니다. 편집기에 액세스하려면 도구 / 옵션 메뉴를 사용하여 고급 섹션을 선택하면 하단에 '구성 편집기'버튼이 나타납니다. 검색 창에 'importexporttoolsng'를 입력하면 모든 확장 환경 설정이 표시됩니다. 그런 다음 필요에 따라 아래의 다양한 환경 설정을 편집 할 수 있습니다.

importexporttoolsng.export.index_date_custom_format

이 환경 설정은 색인 파일에 사용되는 것과 동일한 전체 형식을 따릅니다. 형식이 지정된 날짜 / 시간 문자열이있는 경우 기본적으로 빈 문자열이며 인덱스 날짜 형식이 변경됩니다.

importexporttoolsng.export.attachments.filename_extended_format

importexporttoolsng.export.embedded_attachments.filename_extended_format

이러한 기본 설정은 모두 내 보낸 메시지 파일 이름과 동일한 유형의 확장 파일 이름 형식을 사용합니다. 첫 번째는 외부 첨부 파일을위한 것이고 두 번째는 포함 된 (인라인) 첨부 파일을위한 것입니다.

importexporttoolsng.experimental.use_delivery_date

일반적으로 Thunderbird의 메시지에 대한 '날짜'는 발신자의 메일 클라이언트가 설정 한 날짜입니다. 이 옵션을 'true'로 설정하면 수신자의 메일 클라이언트에서받은 날짜를 사용합니다. 이 날짜는 파일 이름과 색인 파일의 날짜 모두에 사용됩니다.

기본 글로벌 핫키 (바로 가기) 기능

importexporttoolsng.experimental.hot_keys

이 새로운 환경 설정을 통해 사용자는 최대 10 개의 키보드 단축키를 설정할 수 있습니다. 바로 가기는 대부분의 ImportExportTools NG 기능과 기타 Thunderbird 기능을 수행 할 수 있습니다. 환경 설정 문자열은 JSON 형식의 오브젝트로 형식화됩니다. 형식과 키는 다음과 같습니다.

단축키 항목 예 (JSON 객체 배열)

id 키 인덱스 (1-10)
key 키 (단일 문자)
keycode - 'VK_ *'defind 키 코드
키 또는 키 코드가 포함될 수 있지만 둘다는 아닙니다.
modifiers -키 수정 자, 공백으로 구분 된 목록 ( 'control', 'shift', 'alt', 'accel')
oncommand 직접 명령 문자열
contexts 키가 활성화 된 창 컨텍스트
'all' all'-메신저, 작성, 검색중인 모든 창
'messenger' -메인 메신저 창과 그 후손
'compose' 메시지 작성 창
'search' 검색 대화 상자
